Main loop Waits for bytes[] array to fill up with xbee data xbee data consists of a series of packets each packet is 9 bytes in size first 4 bytes is an integer defining radius next 4 bytes is an integer defining theta in degrees last byte is 'a' or 'c', defining ANTICLOCKWISE or CLOCKWISE direction

If less than arcs recieved from xbee then run trace_arc

Radius is calculated as integer using 1st 4 bytes of the packet Radius is obtained in cm

Theta is calculated as integer using next 4 bytes of packet Theta is obtained in degrees

Direction is set using last (9th) byte of packet
